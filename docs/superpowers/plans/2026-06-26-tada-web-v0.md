# Tada Web v0 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. **This plan is built by an agent team** (Architect → Frontend ∥ Backend → Reviewer); see §Execution. **Phase 0 is the gate** — its contracts are *law*; later lanes build against them and never silently change a frozen signature.

**Goal:** Build Tada Web v0 — a capture-first AI to-do app (Todoist flow + native Tada model) with a "do it for me" execution layer and a text/voice chat agent, on Next.js + Neon + the Vercel AI SDK (Gemini).

**Architecture:** Deterministic to-do spine (model never in the list's hot path) + a thin AI layer: extraction = one `generateObject` call; "do it for me" dispatches on `actionType` (meetings/reminders deterministic, research = the only agent loop); one executor fn per capability, called directly by the tap path and wrapped as a gated AI SDK tool for chat/voice. One flat tagged pool; `All` is the only add surface; everything else is a read-only filter-View; one level of subtasks.

**Tech Stack:** Next.js (App Router, RSC, TS-only) · Neon serverless Postgres (+pgvector) · Prisma · Auth.js (NextAuth) Google OAuth · Vercel Blob · Vercel AI SDK (`ai` + `@ai-sdk/google`, Gemini 2.5 Flash/Pro) · OpenAI Realtime (voice, WebRTC) · Postmark (inbound email) · Vercel hosting (`gettada.app`).

## Global Constraints
- **TypeScript only — no Python.** One repo, one deploy.
- **NO Claude/Anthropic in the product runtime** (cost). Gemini for all image+text; OpenAI Realtime for voice only.
- **Never auto-execute a side effect.** Every write shows its concrete effect; fires only on explicit tap or a confirmed (approved) agent tool-call.
- **Capture-first invariant:** persist a `Capture` + a plain `Todo` *before* calling the extractor (all sources). A failed extraction still leaves a usable todo.
- **Flow invariants (from native, do not violate):** one flat tagged pool; `All` is the only add surface; all other views are pure/deterministic filters given `now`; one missing essential field → a single inline ask, not a form; provider seams (extractor/voice/executors) stay behind interfaces.
- **Accent = rust `#c8632e`** on cream `#f0ece3` / graphite `#1b1a18`; auto dark via `prefers-color-scheme`. EB Garamond (display) · Geist (body) · Geist Mono · Caveat (wordmark). Active state = accent or soft raised surface, **never a heavy black fill**.
- **Wire keys snake_case; TS fields camelCase.**
- **Gemini env:** pass `GEMINI_API_KEY` explicitly to `createGoogleGenerativeAI({ apiKey })` (the SDK's default env name is `GOOGLE_GENERATIVE_AI_API_KEY`).
- **Plans:** `free` (50 credits/mo) · `pro` (2000) · `unlimited` (∞, admin-only via `ADMIN_EMAILS`). Credits: `extractTodos:1, chatTurn:1, deepResearch:10`.
- **Spec:** `docs/superpowers/specs/2026-06-24-tada-web-v0-design.md` is the source of truth. Contract refs: `prep/proposed-contracts.md`, `prep/native-flow-contract-reference.md`, `prep/clawdia-port-manifest.md`, `prep/neon-setup.md`.

## Frozen Contracts (Phase 0 ratifies these — then they are LAW)

The TS interfaces in **`prep/proposed-contracts.md`** and the behavioral signatures in **`prep/native-flow-contract-reference.md`** are **ratified as-is** with the settlements below baked in. They are reproduced/extended here as the single frozen reference; do not diverge without the Architect's sign-off.

### Settlements baked into the contracts
- Extractor impl = **AI SDK `generateObject`** (Gemini 2.5 Flash) with a **Zod** schema mirroring `ExtractorOutput`; image passed as `{type:'file', mediaType:'image', data}`. (Not raw `generateContent`.)
- Inbound email provider = **Postmark**; `INBOUND_DOMAIN = "in.gettada.app"`.
- Research = **Gemini 2.5 Pro** agent loop (`ToolLoopAgent`/`generateText` + `tool()` + `stopWhen`), background async, progress → job row / SSE. No workflow engine.
- Voice = **OpenAI Realtime over WebRTC**; routes `/api/voice/{session,tool,usage}`.

### Gaps filled (these complete the frozen set)

// Accounts & auth
type Plan = 'free' | 'pro' | 'unlimited';
interface User { id: string; email: string; name?: string|null; image?: string|null; plan: Plan; createdAt: string; }

interface InviteCode { code: string; maxUses: number; usedCount: number; expiresAt?: string|null; invitedEmail?: string|null; }
async function redeemInvite(code: string, email: string): Promise<boolean>; // atomic conditional UPDATE … RETURNING
function isAdminEmail(email: string): boolean;            // email ∈ ADMIN_EMAILS (comma-split env)
// Auth.js signIn callback: existing user → admit; new user → isAdminEmail ⇒ create w/ plan='unlimited';
//   else redeemInvite(cookieCode, email) must be true, else reject. Google: access_type=offline, prompt=consent;
//   persist refresh_token on the account; refresh against oauth2.googleapis.com ourselves.

// Plan gating
type Capability = 'extractTodos' | 'chatTurn' | 'deepResearch';
const COST: Record<Capability, number> = { extractTodos: 1, chatTurn: 1, deepResearch: 10 };
const PLAN_CREDITS: Record<Plan, number> = { free: 50, pro: 2000, unlimited: Infinity };
class QuotaError extends Error { status = 402; }          // vs 429 for burst
async function withQuota<T>(user: UserCtx, cap: Capability, run: () => Promise<T>): Promise<T>;
// ai_usage PK (user_id, period 'YYYY-MM'); atomic: UPDATE … WHERE used+cost<=limit RETURNING. Admin short-circuits.
// deepResearch: reserve up front, refund on failure.

// Store seam (Prisma-backed; one owner per entity)
interface TadaStore {
  listTodos(userId: string): Promise<Todo[]>;
  createTodo(userId: string, t: Partial<Todo>): Promise<Todo>;
  updateTodo(userId: string, id: string, patch: Partial<Todo>): Promise<Todo>;
  reorderTodo(userId: string, id: string, beforeId?: string|null, afterId?: string|null): Promise<Todo>; // fractional index
  subtasks(userId: string, parentId: string): Promise<Todo[]>;        // children where parentId = id
  createCapture(userId: string, c: Partial<Capture>): Promise<Capture>;
  labels(userId: string): Promise<TodoLabel[]>;  upsertLabelByName(userId: string, name: string): Promise<TodoLabel>;
  views(userId: string): Promise<SavedView[]>;   saveView(userId: string, v: Partial<SavedView>): Promise<SavedView>;
}

// Design tokens: vendor Clawdia's tokens.ts + tokens.generated.css + tokens/*.css + styles.css VERBATIM
//   (no accent swap — sienna #c8632e IS our rust). All tokens = CSS custom properties.

### Front↔back API surface (frozen route list)
| Route | Purpose | Notes |
|---|---|---|
| `POST /api/capture` | image/text capture → capture-first → `extract()` | returns created todos; `withQuota(extractTodos)` |
| `POST /api/inbound/email` | Postmark webhook → verify sig → alias→user → capture-first → extract | server-side, async |
| `POST /api/todos` · `PATCH /api/todos/:id` · `POST /api/todos/:id/reorder` | todo CRUD + drag | deterministic, no LLM |
| `POST /api/todos/:id/finish` | dispatch on `actionType` (tap path) | calls executor directly |
| `POST /api/research` | start `deepResearch` background job | returns job id; progress via SSE/poll |
| `GET /api/research/:id` (or SSE) | research progress / result | writes markdown into `todo.detail` |
| `POST /api/chat` | AI SDK `useChat` endpoint (text agent) | tool registry; gated writes; `withQuota(chatTurn)` |
| `POST /api/voice/session` · `POST /api/voice/tool` · `POST /api/voice/usage` | OpenAI Realtime relay | shared AgentTool registry |
| `POST /api/waitlist` | landing waitlist capture | idempotent on normalized email |
| Auth.js routes (`/api/auth/*`), `/join` | OAuth + invite redemption | per Auth.js |
